## 📖 Overview

**YouTube to Blog Post** is a fully automated AI agent pipeline that converts any YouTube video into a structured, publish-ready blog post — with zero manual steps. Paste a URL, click a button, and within about 60–90 seconds you have a 600–900 word blog post ready to publish.

Built for **content creators, writers, and marketers** who want to repurpose video content at scale without spending hours transcribing, researching, and writing manually. The pipeline handles everything: audio download, transcription, content research, and writing — producing output that reads like it was written by a human, not generated by a template.

## ⚙️ How It Works

```
YouTube URL
    │
    ▼
┌─────────────────────────────────┐
│  TranscriptAgent                │  yt-dlp downloads audio
│  agents/transcript_agent.py     │  Groq Whisper transcribes
└─────────────────────────────────┘
    │  plain-text transcript
    ▼
┌─────────────────────────────────┐
│  BlogPostAgent                  │  LLM Call 1: research pass
│  agents/blog_post_agent.py      │  LLM Call 2: writing pass
└─────────────────────────────────┘
    │  Markdown blog post
    ▼
┌─────────────────────────────────┐
│  Orchestrator                   │  Thin coordinator
│  orchestrator.py                │  No business logic
└─────────────────────────────────┘
    │  { transcript, blog_post }
    ▼
┌─────────────────────────────────┐
│  Streamlit UI                   │  Rendered blog post
│  app.py                         │  Download as .md
└─────────────────────────────────┘
```

### TranscriptAgent
Downloads the audio track of any public YouTube video using `yt-dlp`, converting it to MP3 at 64 kbps via `ffmpeg` to stay within Groq Whisper's 25 MB file limit. The audio is then sent to Groq's `whisper-large-v3-turbo` model, which returns a plain-text transcript. The agent classifies download errors and retries intelligently rather than failing blindly.

### BlogPostAgent
Makes two sequential LLM calls to Groq's `llama-3.3-70b-versatile`. The first call (`_research`) analyses the transcript and produces structured notes — core topic, target audience, key insights, hook angle, section structure, and more. The second call (`_write`) uses those notes alongside the transcript to write a polished, structured blog post. If the research pass fails, the agent falls back to writing directly from the transcript.

### Orchestrator
A thin coordinator with no business logic. Instantiates both agents, calls them in sequence, and returns a dict with both `transcript` and `blog_post` to the Streamlit UI.

## ✨ Features

- 🤖 **Fully automated pipeline** — paste a URL, get a blog post. No intermediate steps.
- 🧠 **Two-stage AI writing** — a dedicated research pass before writing produces significantly better output than a single-prompt approach.
- 🔁 **Intelligent error classification** — errors are classified and handled appropriately: immediate stop for bot detection, exponential backoff for rate limits, fallback for research failures.
- 🍪 **YouTube bot detection handling** — cookie-based bypass supported via the `YOUTUBE_COOKIES` environment variable.
- 📊 **Live progress log** — the UI shows each pipeline step as it runs.
- ⬇️ **Download as .md** — one-click download of the finished blog post.
- 💸 **100% free to run** — Groq's free tier covers normal usage with no credit card required.

## 🚀 Local Setup

### Prerequisites

- Python 3.12
- Git
- ffmpeg (see install commands below)

**Install ffmpeg:**

```bash
# macOS
brew install ffmpeg

# Ubuntu / Debian
sudo apt-get install -y ffmpeg

# Windows (via Chocolatey)
choco install ffmpeg

# Windows (manual)
# Download from https://ffmpeg.org/download.html and add to PATH
```

### Steps

**1. Clone the repository**
```bash
git clone https://github.com/yourusername/yt-agent2.git
cd yt-agent2
```

**2. Create a virtual environment**
```bash
python -m venv venv

# Activate — macOS/Linux
source venv/bin/activate

# Activate — Windows
venv\Scripts\activate
```

**3. Install dependencies**
```bash
pip install -r requirements.txt
```

**4. Set up environment variables**
```bash
cp .env.example .env
```

**5. Add your Groq API key**

Open `.env` and set:
```
GROQ_API_KEY=your_groq_api_key_here
```

Get a free key at [console.groq.com](https://console.groq.com) — no credit card required.

**6. Run the app**
```bash
streamlit run app.py
```

Open [http://localhost:8501](http://localhost:8501) in your browser.

## 🚂 Railway Deployment

**1. Push to GitHub**
```bash
git add .
git commit -m "initial commit"
git push origin main
```

**2. Connect Railway to your GitHub repo**
- Go to [railway.app](https://railway.app) → New Project → Deploy from GitHub
- Select your `yt-agent2` repository

**3. Add environment variables**
- In your Railway project → **Variables** tab → add:
  ```
  GROQ_API_KEY = your_groq_api_key_here
  ```

- Railway automatically detects `nixpacks.toml` and installs `ffmpeg` + `python312`
- `railway.toml` sets the start command — no manual configuration needed
- `PYTHONPATH=/app` is set automatically via `railway.toml` to resolve module imports

Railway will provide a public URL once the deployment is healthy.

## 🍪 YouTube Bot Detection

### The problem

YouTube flags requests from datacenter IP ranges (including Railway's servers) as bots and returns:

```
Sign in to confirm you're not a bot. This helps protect our community.
```

This is a known issue with `yt-dlp` on cloud platforms and is unrelated to the code.

### The fix

Export your browser cookies from an active YouTube session and pass them to `yt-dlp`.

**Step 1 — Install the extension**

Install [Get cookies.txt LOCALLY](https://chromewebstore.google.com/detail/get-cookiestxt-locally/cclelndahbckbenkjhflpdbgdldlbecc) for Chrome (or an equivalent for Firefox).

**Step 2 — Export cookies**

- Go to [youtube.com](https://youtube.com) while signed in to a Google account
- Click the extension → **Export** → save as `cookies.txt`

**Step 3 — Set the environment variable in Railway**

- Railway project → **Variables** → add:
  ```
  YOUTUBE_COOKIES = /path/to/cookies.txt
  ```
  Or paste the raw content of `cookies.txt` directly as the variable value.

The agent will automatically detect and use the cookies on the next request.

## 🔬 How the Agents Work

### TranscriptAgent

The agent tries multiple yt-dlp **player clients** (`android`, `ios`, `tv_embedded`, `web`) in order of reliability. It also sends a real browser `User-Agent` header to reduce bot-detection likelihood without requiring cookies.

**Error classification table:**

| Error Type | Behaviour |
|---|---|
| `bot_detected` | Stop immediately — show actionable cookie instructions |
| `video_unavailable` | Stop immediately — private, deleted, or geo-restricted |
| `ffmpeg_error` | Stop immediately — system dependency missing |
| `rate_limit` | Retry up to 3× with exponential backoff (2s, 4s) |
| `network_error` | Retry up to 3× with exponential backoff |
| `format_error` | Retry once, then stop |
| `unknown` | Retry up to 3× |

### BlogPostAgent

| Step | LLM Call | Model | Temp | Max Tokens | Purpose |
|---|---|---|---|---|---|
| 1 | `_research()` | llama-3.3-70b-versatile | 0.4 | 1500 | Analyse transcript, produce structured notes |
| 2 | `_write()` | llama-3.3-70b-versatile | 0.7 | 2000 | Write 600–900 word blog post from notes |

The research pass covers: core topic, target audience, key insights, hook angle, section structure, human element, and closing direction. If the research call fails for any reason, the agent falls back to writing directly from the transcript — the pipeline never crashes due to a research failure.

For long videos, the transcript is truncated intelligently: 80% from the start, 20% from the end, with a `[...middle omitted...]` marker.

## ⚠️ Limitations

| Limitation | Detail |
|---|---|
| YouTube bot detection | Cloud IPs get flagged — see [cookie fix](#-youtube-bot-detection) above |
| File size limit | Groq Whisper accepts up to 25 MB — approximately 30 minutes of audio at 64 kbps |
| Language | Optimised for English — remove `language="en"` from `_transcribe_with_retry()` for other languages |
| Groq free tier | ~30 requests/minute; longer videos with large transcripts may hit limits |
| Private videos | Cannot download private, age-restricted, or region-blocked videos |
